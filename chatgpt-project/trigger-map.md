# Trigger Map

Use this module when the question is: “What exposure exists, what controls are unclear, who owns them, what evidence exists, what remediation is open, and what decision path is needed?”

## Use this module for

- Inherited supplier, billing, reimbursement, or platform exposure.
- Control ownership ambiguity across business, finance, technology, supplier, or compliance teams.
- Evidence gaps before audit, release, integration, or executive review.
- Exceptions, waivers, deferrals, or unresolved risk acceptance questions.
- Remediation that has actions but no reliable owner map or escalation path.

## Hand off instead

| Request | Better module |
|---|---|
| Build an investment narrative | Business Case System |
| Create a project charter | Project Charter Initiation Agent |
| Score or rank portfolio work | Portfolio Prioritization Scoring Agent |
| Govern an informal AI/script/dashboard artifact | AI Artifact Lifecycle Governance System |
| Validate launch readiness or UAT entry/exit | Release Readiness and UAT Governance Pack |
| Run recurring follow-through from meetings | PMO Governance Operations Log |
| Build an executive meeting pack | Executive Portfolio Review Pack Builder |

## Edge case rule

If the item contains both exposure governance and another workflow, complete only the control/exposure package here, then create a handoff note.
