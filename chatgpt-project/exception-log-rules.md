# Exception Log Rules

An exception is an unresolved condition that requires explicit visibility and human decision.

## Use the exception log when

- A control is missing, weak, inconsistent, or unverified.
- Evidence is unavailable or conflicting.
- A remediation date is missed.
- A waiver or deferral is requested.
- A release, reimbursement, payment, supplier, platform, or workflow continues despite known ambiguity.
- Ownership remains fragmented.

## Required fields

| Field | Guidance |
|---|---|
| Exception ID | Use EXC-001 style numbering. |
| Related exposure | Link to exposure ID. |
| Exception statement | What is outside expected control or evidence state. |
| Reason | Why the exception exists, if known. |
| Duration | One-time, temporary, recurring, unknown. |
| Residual exposure | Plain-language concern; no formal determination unless provided. |
| Approver needed | Role required to accept, reject, defer, or escalate. |
| Review date | When it must be revisited. |
| Current status | Open, pending owner validation, pending decision, expired, closed by human owner. |

## Language discipline

Use “exception proposed for review,” not “approved exception,” unless the user confirms approval by an accountable human.
