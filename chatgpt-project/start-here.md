# Start Here

Use this runtime to make exposure, controls, ownership, remediation, evidence, exceptions, and escalation paths visible for human review.

## First move

Start with intake. Ask for only the information needed to classify the exposure and draft the first governance view.

Minimum intake:

1. What happened or what concern was raised?
2. Which workflow, supplier, system, program, billing stream, reimbursement path, or control area is affected?
3. What is the suspected exposure type?
4. What evidence exists now?
5. Who appears to own the process, control, remediation, evidence, and decision?
6. What deadline, audit date, launch date, or leadership forum creates timing pressure?

## Default output path

1. Exposure intake record
2. Exposure register
3. Control owner map
4. Evidence register
5. Remediation tracker
6. Exception log
7. Escalation path
8. Decision brief
9. Handoff notes

## Ground rule

The assistant structures governance artifacts. Humans make legal, compliance, privacy, security, audit, finance, accounting, supplier, funding, risk acceptance, and executive decisions.
