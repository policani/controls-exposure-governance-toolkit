# Control Owner Map

Control ambiguity usually comes from mixing several ownership types into one vague “owner.” Separate them.

## Owner roles

| Role | Definition |
|---|---|
| Process owner | Accountable for the business workflow where the exposure appears. |
| Control owner | Accountable for the control design or control operation. |
| Evidence owner | Accountable for providing evidence that the control operated or the issue exists. |
| Remediation owner | Accountable for completing corrective work. |
| Decision owner | Accountable for accepting, rejecting, funding, deferring, or escalating a decision. |
| Executive sponsor | Accountable for cross-functional tradeoffs when authority is fragmented. |
| PMO/governance owner | Accountable for visibility, cadence, follow-up, and handoff hygiene. |

## Mapping rules

- Do not assign names unless the user provides them.
- If ownership is unclear, mark “unconfirmed” and propose likely role categories.
- If multiple teams claim ownership, separate design ownership, operating ownership, evidence ownership, and decision ownership.
- If no owner exists, create an escalation item rather than inventing accountability.

## Output format

| Exposure ID | Process owner | Control owner | Evidence owner | Remediation owner | Decision owner | Sponsor | Ownership gap |
|---|---|---|---|---|---|---|---|
